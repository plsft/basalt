# LinkedIn — launch announcement

Today I'm publishing Basalt — a tool I've been building for the last few months
to keep my own thinking honest.

The problem it solves is specific. If you write a lot — strategy memos,
research notes, build logs, transcript-of-yourself thinking — your second brain
becomes a corpus you can't hold in your head. Search tools tell you *what's
there*. Basalt tells you *what you actually believe but never wrote down*.

It does this by reading your Markdown vault and producing one short Brief
per week: a paragraph about the through-line you've been circling, the places
you're quietly disagreeing with yourself, the projects whose meaning has
slid sideways, and the notes you wrote months apart that are secretly the
same idea.

For me — running fractional engagements where I need to be coherent across
five different client contexts — it's been the difference between feeling
like I'm losing the thread and being able to point at a paragraph and say
"this is the actual through-line of my work this quarter."

Basalt is:
- Local-first (the Open tier never makes a network call)
- MIT-licensed (the engine is open-source; Pro is a hosted convenience on top)
- Read-only on your notes (it never modifies a file; an architectural test in the codebase enforces this)
- Five surfaces from one engine (CLI, Obsidian plugin, MCP server, native desktop, web cockpit)

If you're a writer, a strategist, a researcher, or anyone who maintains a
substantial Markdown vault — try it.

basalted.com
