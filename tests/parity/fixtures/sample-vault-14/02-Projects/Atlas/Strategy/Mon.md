# Atlas — Launch Sequencing (Monday)

> The launch sequencing is finally settled.

## Decision

After three weeks of going back and forth, the call is: **ship the product first, then write the docs once the surface area stabilizes.** Documentation written ahead of a moving target ages worse than no documentation.

The order is product → internal usage → docs → external launch. Mentor agreed in [[Mentor]]. We keep moving.

## Implications

- Engineering focuses entirely on the core verb library through May
- Documentation effort starts in June, after the API surface is stable
- External launch in July or August
- This sequencing is settled. Stop relitigating it.
